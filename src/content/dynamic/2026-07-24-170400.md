**题目：**

Linux 系统中，命令 `date` 可查看当前时间，为了方便在 Shell 脚本中调用时间，需要将该命令定义成变量 `TIME`，以下哪个选项为正确的定义方式？

A. `TIME=date`

B. `TIME=$date`

C. `TIME=${date}`

D. `TIME=$(date)`

> **答案：D**

---

# 一、题目考查的知识点

本题考查的是 **Shell 的命令替换（Command Substitution）**。

所谓命令替换，就是：

> **执行一个命令，并把该命令的输出结果作为字符串返回。**

Shell 中有两种写法：

```bash
TIME=$(date)
```

或者旧式写法：

```bash
TIME=`date`
```

其中：

* `date`：执行 date 命令，输出当前时间。
* `$(...)`：表示执行括号中的命令，并取得其输出结果。
* `TIME=`：将输出结果赋值给变量。

例如：

```bash
TIME=$(date)

echo $TIME
```

输出可能为：

```text
Thu Jul 24 16:30:51 CST 2026
```

因此 **TIME 保存的是 date 命令执行后的结果，而不是命令本身。**

---

# 二、逐项分析

## A．TIME=date

```bash
TIME=date
```

这里没有使用命令替换。

Shell 会直接把字符串 `"date"` 赋值给变量。

因此：

```bash
echo $TIME
```

输出

```text
date
```

而不是

```text
Thu Jul 24 16:30:51 CST 2026
```

所以 **错误。**

---

## B．TIME=$date

```bash
TIME=$date
```

这里的 `$` 表示**变量扩展（变量引用）**。

例如：

```bash
NAME=Tom

echo $NAME
```

输出：

```text
Tom
```

因此：

```bash
$date
```

表示的是：

> **读取变量 date 的值**

而不是执行命令。

如果系统中没有：

```bash
date=...
```

那么：

```bash
echo $date
```

输出为空。

所以：

```bash
TIME=$date
```

等价于

```bash
TIME=
```

因此错误。

---

## C．TIME=${date}

```bash
TIME=${date}
```

`${}` 也是变量扩展。

它只是

```bash
$date
```

的另一种、更规范的写法。

例如：

```bash
name=Tom

echo ${name}
```

输出：

```text
Tom
```

它仍然表示：

> **读取变量 date 的值**

并不会执行 `date` 命令。

因此也是错误。

---

## D．TIME=$(date)

```bash
TIME=$(date)
```

这里使用的是 **命令替换**。

Shell 的执行过程如下：

### 第一步

先执行命令：

```bash
date
```

得到：

```text
Thu Jul 24 16:30:51 CST 2026
```

---

### 第二步

Shell 将命令输出替换进去：

```bash
TIME="Thu Jul 24 16:30:51 CST 2026"
```

因此：

```bash
echo $TIME
```

输出：

```text
Thu Jul 24 16:30:51 CST 2026
```

符合题意。

因此 **D 正确。**

---

# 三、三个最容易混淆的符号（重点）

考试经常考下面三个：

| 写法      | 含义                   | 是否执行命令 |
| ------- | -------------------- | ------ |
| `$(命令)` | **命令替换**，运行命令并取得输出   | ✅ 是    |
| `${变量}` | **变量扩展**，读取变量值（推荐写法） | ❌ 否    |
| `$变量`   | **变量扩展**的简写，读取变量值    | ❌ 否    |

例如：

```bash
NAME=Tom

echo $NAME
```

输出

```text
Tom
```

而

```bash
echo $(date)
```

则输出当前时间。

因此一定要记住：

> **`$()` 用于执行命令；`${}` 和 `$变量` 用于读取变量。**

---

# 四、拓展考点

## ① 命令替换只会捕获标准输出（stdout）

这是 Shell 中非常重要的一点。

例如：

```bash
TIME=$(date)
```

Shell 实际上会捕获：

> **标准输出（stdout）**

而不会捕获：

> **标准错误（stderr）**

例如：

```bash
OUT=$(ls not_exist)
```

终端会显示：

```text
ls: cannot access 'not_exist': No such file or directory
```

但是：

```bash
echo "$OUT"
```

输出为空。

原因就是：

* 错误信息输出到了 **stderr**
* `$(...)` 默认只保存 **stdout**

---

如果希望把错误信息也保存进去，可以进行重定向：

```bash
OUT=$(ls not_exist 2>&1)
```

其中：

```text
2
```

表示 stderr，

```text
1
```

表示 stdout，

```text
2>&1
```

表示：

> **把标准错误重定向到标准输出。**

这样：

```bash
echo "$OUT"
```

就会得到：

```text
ls: cannot access 'not_exist': No such file or directory
```

这是 Linux 运维、Shell 编程中非常经典的知识点。

---

## ② `$()` 可以嵌套

例如：

```bash
echo $(dirname $(pwd))
```

执行顺序：

1. `pwd`
2. `dirname`
3. `echo`

这种写法可无限嵌套。

而旧式反引号：

```bash
`date`
```

由于反引号本身需要转义，嵌套非常麻烦，因此现代 Shell 都推荐使用：

```bash
$(...)
```

---

## ③ 命令替换得到的是字符串

例如：

```bash
NUM=$(expr 3 + 5)

echo $NUM
```

得到

```text
8
```

注意：

变量中保存的是字符串 `"8"`，而不是数值类型。

Shell 本身没有严格的数值类型，变量默认都是字符串，需要运算时再进行算术扩展（如 `$(())`）。

---

# 五、知识点总结

### ① 三种 `$` 的用法

| 写法      | 名称                         | 作用             |
| ------- | -------------------------- | -------------- |
| `$(命令)` | 命令替换（Command Substitution） | 执行命令，并获取输出结果   |
| `${变量}` | 变量扩展（Parameter Expansion）  | 获取变量值，推荐写法     |
| `$变量`   | 变量扩展简写                     | 获取变量值，适用于简单变量名 |

---

### ② 本题四个选项总结

| 选项             | 实际含义                  | 是否正确 |
| -------------- | --------------------- | ---- |
| `TIME=date`    | 把字符串 `"date"` 赋值给变量   | ❌    |
| `TIME=$date`   | 读取变量 `date` 的值        | ❌    |
| `TIME=${date}` | 读取变量 `date` 的值        | ❌    |
| `TIME=$(date)` | 执行 `date` 命令，并将输出赋给变量 | ✅    |

---

## 一句话记忆

看到：

* **`$(...)`** → **执行命令，拿输出。**
* **`${...}`** → **读取变量值。**
* **`$变量`** → **`${变量}` 的简写。**

> **口诀：** **小括号执行命令，花括号读取变量；命令替换抓 stdout，不抓 stderr。**
