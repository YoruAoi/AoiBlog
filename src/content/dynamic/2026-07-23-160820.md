**每日一题**

### 题目
在openEuler系统中运行了以下脚本：
```bash
#!/bin/bash
#NUM=5
"NUM"=6
NUM=7
echo $NUM
```
以下哪一项为输出的NUM的值？
- A.0
- B.6
- C.7
- D.5

正确答案是 **C. 7**。

下面逐行分析这段 Bash 脚本。

```bash
#!/bin/bash
#NUM=5
"NUM"=6
NUM=7
echo $NUM
```

---

## 第1行

```bash
#!/bin/bash
```

这是**Shebang（释伴行）**，格式：**#!解释器路径**
- #! 组合符号英文名称 shebang（也叫 hashbang）。
- 作用：告诉操作系统使用哪一个程序来解析执行当前脚本。题目表示使用 `/bin/bash` 来解释执行脚本。
与变量无关。

---

## 第2行

```bash
#NUM=5
```

这一行前面有 `#`。

在 Shell 中：

* `#` 后面的内容都是**注释**
* 注释不会执行

因此

```bash
NUM=5
```

实际上**没有执行**。

此时：

```
NUM 还不存在
```

---

## 第3行

```bash
"NUM"=6
```

这一行很多同学都会误认为给变量赋值，其实**不是**。

### Bash 中变量赋值必须满足：

```bash
变量名=值
```

例如

```bash
NUM=6
```

变量名必须是：

```
NUM
```

不能写成

```bash
"NUM"
```

因为：

```bash
"NUM"
```

表示的是**一个字符串**。

Bash 会把整行解释成：

> 执行一个名字叫 `"NUM"=6` 的命令

而不是变量赋值。

实际上执行时会报错，例如：

```text
"NUM"=6
bash: NUM=6: command not found
```

但是：

**脚本默认不会因为这一句报错而停止。**

如果没有：

```bash
set -e
```

那么后面的语句仍继续执行。

因此：

这一句**没有给 NUM 赋值**。

---

## 第4行

```bash
NUM=7
```

这才是**真正合法的变量赋值**。

现在

```
NUM = 7
```

---

## 第5行

```bash
echo $NUM
```

`$NUM`

表示取变量 NUM 的值。

于是输出：

```text
7
```

---

# 为什么不是6？

因为

```bash
"NUM"=6
```

不是变量赋值。

正确写法只能是：

```bash
NUM=6
```

Shell 对变量赋值要求非常严格：

* **变量名不能加 `$`**
* **变量名不能加引号**
* **等号两边不能有空格**

例如：

正确：

```bash
NUM=6
name=Tom
```

错误：

```bash
"NUM"=6      ❌
'NUM'=6      ❌
$NUM=6       ❌
NUM =6       ❌
NUM= 6       ❌
NUM = 6      ❌
```

---

# 如果第3行改成

```bash
NUM=6
NUM=7
echo $NUM
```

由于 Bash 后面的赋值会覆盖前面的赋值：

```
NUM=6
↓↓
NUM=7
```

最后仍然输出

```
7
```

---

## 本题考点总结

1. `#` 开头是注释，不执行。
2. Bash 变量赋值格式必须是：

   ```bash
   变量名=值
   ```
3. `"NUM"=6` 是非法赋值，会报 `command not found`，不会修改变量。
4. 后面的合法赋值 `NUM=7` 生效。
5. `echo $NUM` 输出变量的值，因此最终输出：

**7（选 C）**。
